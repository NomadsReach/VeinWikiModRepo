Introduction to Modding VEIN

Note: This is a living document that will be updated as new modding techniques are discovered. If you create tutorials or figure out new methods, please share them with the community via Discord.

Step 1: Introduction and Key Information

▼

Modding VEIN is a complex process if you haven't modded games before or explored Unreal Engine game files. Make sure you have caffeine on hand and an hour or two to learn the ins and outs. This guide is designed to be as beginner-friendly as possible, but it's still complicated. Once you complete the process once or twice, you'll get the hang of it.

Links for every tool mentioned are located throughout this document and compiled at the bottom.

Important: This tutorial requires downloading multiple programs from various sources including GitHub. If this concerns you, this may not be the tutorial for you.

Community Support

If you need help:

  • Join the Modding Discord: https://discord.gg/nsaPf7wAkP
  • Ask in the Modding section of the main VEIN Discord: https://discord.gg/vein-survival

Step 2: Downloads and Important Set-Up

▼

You will need all of these programs for basic VEIN modding:

Required Downloads

  • 7-Zip - File compression utility
    https://www.7-zip.org/download.html
  • FModel - Asset browser (use most recent version)
    https://github.com/4sval/FModel/releases
  • VEIN Mapping File - Required .usmap file
    Download from Google Drive
  • UAssetGUI - Asset editor
    https://github.com/atenfyr/UAssetGUI/releases
  • UnrealPak - Packaging tool
    https://github.com/Dmgvol/UE_Modding/raw/main/Tools/UnrealPak.zip

Step 3: Setting Up FModel

▼

Extract and Run FModel

Unzip and run FModel.exe.

Note: If prompted, install .NET 8.0 or later from Microsoft's website.

Configure Game Detection

Make sure "Detected Game" shows VEIN.

If VEIN is not available:

  • Click "Add Undetected Game"
  • Set Name: VEIN
  • Set Directory to: C:\Program Files (x86)\Steam\steamapps\common\Vein\Vein\Content\Paks

Click OK

FModel will load the game's files, but we need to teach it how to read them using the mappings file.

Configure Settings

  1. Click "Settings" in the top bar
  2. Under "General", enable "Local Mapping File"
  3. Choose the Mapping File Path (the .usmap file you downloaded)
  4. Choose your Output Directory (where exports will be saved - choose somewhere you know)
  5. Under "Models", change Socket Format (ActorX) to "Don't Export Bone Sockets"
  6. Click OK (FModel may restart)

Load Game Files

Once FModel restarts, click "Archives" then click "Load". This will load all files from the game and take you to the "Folders" tab.

Step 4: Setting Up UAssetGUI

▼

Open UAssetGUI

Launch the UAssetGUI application.

Import Mappings

  1. Click Utils → Import mappings...
  2. Import the same .usmap file that you downloaded and applied in FModel

Set Game Version

In the top right of your screen, it should say "VEIN". Set the Unreal version to 5.4.

Step 5: Setting Up UnrealPak

▼

Create Workspace Folder

Make a new folder in a location you can easily find. For this guide, we'll use a folder called "VEINModding".

Extract UnrealPak

Unzip the UnrealPak.zip file into this folder.

Verify Installation

Make sure your folder contains:

  • UnrealPak.exe
  • UnrealPak-With-Compression.bat
  • Supporting DLL files

That's it! Setup complete.

Step 6: Finding Stuff in FModel

▼

In FModel, you can browse to find everything that exists within the game.

If you see a folder that has a "package" in it (shown at the top of the screen under the folders tab), click on the "Packages" tab to see what's in it.

You can then double-click on items to see the code, 3D models, or whatever is included in that package.

Step 7: Mod Tutorial - Changing and Adding Properties

▼

This tutorial walks you through editing properties on items and adding new properties.

We'll recreate the "Easy Lockpicking Mod" to demonstrate all necessary steps.

Locate the Locks in FModel

All lock assets are located in: Vein/Content/Vein/LockTypes

Find Lockpicking Difficulty Property

Double-click on LTD_BasicLock_Simple.uasset. If you don't see anything related to difficulty, try the next one.

Identify the Property

We found PickableEasinessModifier! Note that the "Simple" lock lacks this modifier - this will be important later.

List All Objects to Modify

We need to modify every object with PickableEasinessModifier AND the BASE lock LTD_BasicLock_Simple.

Our final list of exports:

  • LTD_BasicLock_Simple
  • LTD_BasicLock_Tough
  • LTD_BasicLock_VeryTough
  • LTD_BasicLock
  • LTD_BasicLock_CodeLock
  • LTD_BasicLock_EnhancedLock
  • LTD_BasicLock_RemoteLock

Export as Raw Data

Export these as Raw Data (.uasset). They'll be saved to your FModel output directory. A link at the bottom of the screen will take you directly to your exports.

Open First File in UAssetGUI

  1. Open UAssetGUI
  2. Click File → Open
  3. Navigate to your FModel output folder
  4. Open LTD_BasicLock.uasset

Find the Property

Search for PickableEasinessMultiplier. Based on what we saw in FModel, it should be 0.5 for this lock type.

Change the Property

Double-click "0.5" on the PickableEasinessMultiplier line and change it to "10".

Save Changes

Click File → Save.

Note: UAssetGUI saves a backup (*.bak) file. Feel free to delete these or keep them.

Modify Remaining Locks

Adjust all remaining lock types EXCEPT LTD_BasicLock_Simple (it doesn't have the property yet).

Adding a Property (Advanced)

We need to ADD a property to LTD_BasicLock_Simple instead of just changing one.

Changing properties is EASY. Adding properties is COMPLEX.

Step 11.5: Prepare Text Editor

You'll need a text editor that can handle JSON (Notepad++ or VSCode recommended).

Export as JSON

We need a reference (what we want) and the file to modify (what we have) as JSON.

  1. Open LTD_BasicLock_VeryTough in UAssetGUI
  2. Click File → Save As
  3. Name it LTD_BasicLock_VeryTough and set "Save as type" to UAssetAPI JSON (*.json)
  4. Open LTD_BasicLock_Simple in UAssetGUI
  5. Click File → Save As
  6. Name it LTD_BasicLock_Simple and set "Save as type" to UAssetAPI JSON (*.json)
  7. Navigate to where you saved these JSON files and open both
  8. In Notepad++, you can view them side by side

Find Property in Reference

  1. In the VeryTough Lock JSON, search (Ctrl+F) for PickableEasinessMultiplier
  2. Note the surrounding properties:
    • Before: bRemovable
    • After: RequirementsToCopyLock

Locate Insertion Point

In the Simple Lock JSON, search for bRemovable. You'll find it around line 182-190, with RequirementsToCopyLock on lines 191-234. We'll insert our property between these two.

Copy Property Block

From the VeryTough lock, copy this code:

{
  "$type": "UAssetAPI.PropertyTypes.Objects.DoublePropertyData, UAssetAPI",
  "Value": 10.0,
  "Name": "PickableEasinessMultiplier",
  "ArrayIndex": 0,
  "IsZero": false,
  "PropertyTagFlags": "None",
  "PropertyTagExtensions": "NoExtension"
},

Insert Property

  1. In the Simple lock JSON, click on the } on line 190
  2. Press Enter
  3. Paste the code
  4. Make SURE it looks EXACTLY like properties before and after it
  5. Do not miss ANY symbols, commas, or brackets

Verify in UAssetGUI

  1. Save the edited LTD_BasicLock_Simple.json
  2. Open it in UAssetGUI
  3. Verify it now contains the PickableEasinessMultiplier property
  4. If successful, click File → Save As and overwrite the old LTD_BasicLock_Simple.uasset

Create Mod Folder Structure

  1. Go to where you've been saving everything
  2. Delete the .json files
  3. Optional: Delete the .bak backup files
  4. Create a folder with your mod name - must end in _P
    • Example: LxEZLockpicking_P
  5. Inside, create this folder structure:
    • YourMod_P/Vein/Content/Vein/LockTypes/
    • Note: "Vein" appears twice, just like in FModel
  6. Copy ALL of your lock files (.uasset and .uexp) into the LockTypes folder

Remember: Add the important note to your mod if you upload it!

Step 8: Packing Files using UnrealPak

▼

Move Mod Folder

Move your mod folder (the one ending in _P) to where your UnrealPak is located (e.g., "VEINModding" folder).

Drag and Drop

Drag your mod folder (ending in _P) onto UnrealPak-With-Compression.bat.

Step 20-21: Wait for Completion

A command prompt will show packing progress. Press any key when complete to finalize and close the prompt.

Verify .pak Creation

A .pak file will be created in your folder (e.g., LxEZLockpicking_P.pak).

Install Your Mod

Move your mod .pak file to the game's Paks folder:

C:\Program Files (x86)\Steam\steamapps\common\Vein\Vein\Content\Paks

Test

Load the game and test your mod!

Note: There's a quirk with lockpicking where setting the multiplier to 10 makes it so you unlock by moving the pick ALL THE WAY to the right and pressing D, rather than being able to do it anywhere.

Step 9: Setting Up An Unreal Engine Project

▼

Start by installing Unreal Engine 5.4.4 (you can figure this out).

Creating a VEIN Project

  1. Launch Unreal Engine 5.4.4
  2. Select the GAMES category on the left
  3. Select Blank
  4. Set your project location (somewhere with good space)
  5. Set Project Name: Vein
  6. Click Create

Your project now exists! This is what you need for basic Unreal Engine mods. For more complex mods, see the Unreal Engine (Advanced) section.

Step 10: Mod Tutorial - Changing Fonts in VEIN

▼

This tutorial walks you through replacing fonts used in VEIN.

This recreates the "Comic Sans Font Changer" Mod.

Important: You need Unreal Engine 5.4.4 installed and a UE project set up. You also need basic knowledge of navigating Unreal Engine.

Step 1-5: Create Folder Structure

  1. Open your UE Project called "Vein"
  2. Open the content drawer (should be in the "All" section)
  3. Create a new folder called Vein and open it
  4. Create a new folder called UI and open it
  5. Create a new folder called Fonts and open it

Step 6-7: Import Font

  1. Drag your desired font file into the Fonts folder in the content drawer
  2. On the "Font Face Import Options" popup:
    • Check "Apply to All"
    • Click "No"

Step 9-10: Create Font Copies

Copy (Ctrl+C) and paste (Ctrl+V) your font 8 times.

Rename all 8 copies to these exact names:

  • BebasKai
  • FF_Inter
  • FF_Inter-Bold
  • FF_Inter-ExtraBold
  • FF_Inter-ExtraLight
  • FF_Inter-Light
  • Title
  • US101

Cook Fonts

  1. Click Platforms in the top of your screen
  2. Click Windows
  3. Click Cook Content
  4. If you get a popup about missing something, ignore it and proceed

Step 12-13: Find Cooked Content

Navigate to: UnrealEngine Projects\Vein\Saved\Cooked\Windows\Vein\Content\Vein\UI\Fonts

Each font should have .uasset, .uexp, and .ufont files. Double-check they're all there.

Create Mod Folder

  1. Create a folder ending in _P (e.g., LxComicSans_P)
  2. Inside, create: YourMod_P/Vein/Content/Vein/UI/Fonts/
  3. Move all cooked files into the Fonts folder

Step 15-21: Package and Install

  1. Move your mod folder to your UnrealPak directory
  2. Drag folder onto UnrealPak-With-Compression.bat
  3. Wait for packaging to complete
  4. Copy the created .pak file to: C:\Program Files (x86)\Steam\steamapps\common\Vein\Vein\Content\Paks
  5. Load the game and test!

Step 11: Unreal Engine Project (Advanced)

▼

This tutorial covers setting up Unreal Engine for Logic Mods and Blueprint Mods.

Important: You need to set up UE4SS for these advanced mods to work.

Set Project Name

  1. Click Edit → Project Settings
  2. Where it says "Project Name" (likely blank), enter Vein

Disable Io Store and Enable Chunk Generation

  1. Click Edit → Project Settings
  2. On the left, under "Project", find Packaging
  3. Find "Use Io Store" and disable it
  4. Find "Generate Chunks" and enable it

Enable Chunk ID Assignments

  1. Click Edit → Editor Preferences
  2. On the left, under "General", find Experimental
  3. Find "User Interface" dropdown and enable "Allow ChunkID Assignments"

These changes let you package .pak logic mods and blueprint mods from Unreal Engine without using UnrealPak (which can have issues with these mod types).

Step 12: Downloading and Installing UE4SS for Mod Devs

▼

UE4SS is required to load logic mods and blueprint mods.

Download: https://github.com/UE4SS-RE/RE-UE4SS/releases/tag/experimental-latest

Version Required: zDEV-UE4SS_v3.0.1-599-g1a6cdbc

Important: Mods using UE4SS require end users (players) to also install UE4SS in the same version. Make sure to list UE4SS as a requirement when uploading your mod.

Installation Steps

  1. Scroll down on the GitHub page and download zDEV-UE4SS_v3.0.1-599-g1a6cdbc
  2. Go to Steam Library → Right-click VEIN → Manage → Browse Local Files
    • This is your <GAME_ROOT> folder
    • Example: C:\Program Files (x86)\Steam\steamapps\common\Vein
  3. Navigate to: <GAME_ROOT>\Vein\Binaries\Win64\
  4. Extract the zipped folder "UE4SS-VEIN" into the Win64 folder
  5. Verify Win64 folder now contains:
    • D3D12 (folder)
    • DML (folder)
    • Ue4ss (folder)
    • dwmapi.dll
    • tbb.dll
    • tbb.pdb
    • Tbbmalloc.dll
    • Tbbmalloc.pdb
    • Vein-Win64-Test.exe
    • Vein-Win64-Test.pdb
  6. Verify both ue4ss folder and dwmapi.dll are in the same folder as Vein-Win64-Test.exe
  7. Open the game once and close it (this lets UE4SS add itself to your install)
  8. You should now have a new "LogicMods" folder at: <GAME_ROOT>\Vein\Content\Paks
  9. This LogicMods folder is where you put logic mods and blueprint mods

Step 13: Mod Tutorial - Blueprint Modding (Making a Simple Widget)

▼

Prerequisites: Complete the Unreal Engine Project (Advanced) section AND install UE4SS v3.0.1-599-g1a6cdbc. Your mod will not work without both.

This tutorial tests that both UE4SS and Unreal are set up correctly for Logic and Blueprint mods.

Step 1-4: Create ModActor

  1. In Unreal, navigate to your Content folder and create a folder called Mods
  2. Within Mods, create another folder (your mod name - e.g., TestMod)
  3. Open your mod folder
  4. Create ModActor:
    • Right-click → Create Basic Asset → Blueprint Class
    • Click Actor
    • Rename to ModActor

Save

Click File → Save All (get used to saving often).

Step 6-7: Create Widget Blueprint

  1. Right-click → Create Advanced Asset → UI → Widget Blueprint
  2. Click User Widget
  3. Rename starting with WBP_ (standard convention)
    • Example: WBP_TestWidget

Step 8-9: Design the Widget

  1. Double-click your widget blueprint to open it
  2. Familiarize yourself with the editor:
    • Palette (top left) - widget options
    • Hierarchy (bottom left) - widget structure
    • Viewport (center) - visual preview
    • Details (right) - property editor
  3. In Palette, search for "Canvas Panel"
  4. Drag Canvas Panel onto your widget in the Hierarchy
  5. Search for "Text" in Palette
  6. Drag Text onto the Canvas Panel
  7. Click the Text in Hierarchy
  8. In Details panel, find Content → Text
  9. Change text to anything (e.g., "Mods working! Huzzah!")
  10. Move text to center of screen (VEIN's UI blocks top-left)
    • Use Details panel (Slot → Position X & Y) OR
    • Click and drag in Viewport
  11. Customize as desired (e.g., size 64)
  12. Click Compile then File → Save All

Program the ModActor

  1. Open ModActor Blueprint
  2. Click Event Graph tab at the top
  3. Familiarize yourself with Event Graph:
    • Event BeginPlay - executes when actor starts
    • Event ActorBeginOverlap - executes on collision
    • Event Tick - executes every frame (avoid using - causes FPS drops)

Create Widget Logic

  1. Locate "Event BeginPlay"
  2. Drag from the arrow and release in empty space
  3. Type "Construct" and click Construct object from class
  4. Click dropdown under "Class" and select your widget name
  5. Drag from arrow on Construct node and type "Add to Viewport"
  6. Drag from "Return Value" (blue dot) on Construct node to "Target" (blue dot) on Add to Viewport
  7. Click Compile then File → Save All

Assign to Chunk

  1. Open Content Drawer
  2. Highlight ModActor and your widget (Shift+Click)
  3. Right-click → Asset Actions → Assign to Chunk
  4. Choose a number you'll remember (different for each mod)
    • Example: 42
  5. File → Save All

Cook the Project

  1. Click back to your Untitled Scene tab
  2. Click Platforms dropdown at top
  3. Click Windows → Package Project
  4. Choose a packing location (e.g., UnrealEngine Projects\Vein folder)
  5. Wait 30 seconds for packaging to complete

Locate Your Packed .pak

  1. Go to your selected folder
  2. Navigate to: Windows\Vein\Content\Paks
  3. Find your pakchunk file: pakchunk(YOUR_NUMBER)-Windows.pak
    • Example: pakchunk42-Windows.pak
  4. If you also see .ucas or .utoc files, go back to the Advanced setup section - you skipped steps
  5. Rename to your mod name (must match folder name from step 2 exactly)
    • Example: TestMod.pak

Step 15-17: Install and Test

  1. Locate LogicMods folder: <GAME_ROOT>\Vein\Content\Paks\LogicMods
  2. Copy/move your mod .pak to LogicMods folder
  3. Start the game - your text should appear on the main menu!

Success!

Congratulations! You just made your first blueprint mod for VEIN. You can now add an extremely large amount of functionality to the game.

Step 15: Important Note for Mod Uploaders

▼

This is serious! We do not want to burden the devs with extra work!

If you upload your mods ANYWHERE, include this disclaimer:

***IMPORTANT NOTE!!!***

If you experience ANY bugs or glitches while you have this mod, or any others, installed, please make sure to remove the mods and test if the bug continues before reporting it to the devs!!!

We don't want to cause bugs through modding that devs have to spend time solving, only to find out it was because we missed a comma in our code.
